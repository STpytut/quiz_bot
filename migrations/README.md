# Инструкция по применению миграции

## Шаг 1: Примените миграцию базы данных

Для того чтобы анонимные участники могли проходить викторины со своим именем, необходимо добавить колонку `participant_name` в таблицу `quiz_sessions`.

### Вариант 1: Новая установка

Если вы создаете базу данных с нуля:
- Просто выполните весь файл `supabase-schema.sql` в SQL Editor вашего проекта Supabase
- Колонка `participant_name` уже включена в схему

### Вариант 2: Существующая база данных

Если у вас уже есть база данных с данными:
1. Откройте SQL Editor в панели Supabase
2. Выполните следующий SQL:

```sql
ALTER TABLE quiz_sessions 
ADD COLUMN IF NOT EXISTS participant_name TEXT;
```

Или выполните файл `migrations/add_participant_name.sql`

## Что изменилось

### Для участников:
- ✅ Теперь можно проходить викторины без регистрации
- ✅ Достаточно ввести только имя
- ✅ Имя запоминается для повторных прохождений (localStorage)
- ✅ Авторизованные пользователи автоматически пропускают диалог ввода имени

### Для создателей викторин:
- ✅ В статистике появилась таблица "Результаты участников"
- ✅ Видно имя каждого участника и его результат
- ✅ Результаты отсортированы по дате прохождения
- ✅ Цветовая индикация успешности (зеленый/желтый/красный)

## Проверка

После применения миграции:
1. Откройте публичную викторину в режиме инкогнито
2. Убедитесь, что появился диалог ввода имени
3. Пройдите викторину
4. Проверьте, что имя появилось в статистике

## Структура таблицы quiz_sessions

```sql
quiz_sessions (
  id UUID PRIMARY KEY,
  quiz_id UUID,
  user_id UUID NULL,           -- NULL для анонимных участников
  participant_name TEXT NULL,   -- Имя анонимного участника
  score INTEGER,
  total_questions INTEGER,
  completed_at TIMESTAMP,
  created_at TIMESTAMP
)
```
