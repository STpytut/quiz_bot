# QuizMaster - Платформа для создания викторин

Современное одностраничное приложение для создания и проведения интерактивных викторин с обратной связью в реальном времени и статистикой.

## Возможности

- ✅ Создание неограниченного количества викторин с множеством вопросов
- ✅ Множественный выбор ответов с подсветкой правильного/неправильного в реальном времени
- ✅ Публичные и приватные викторины
- ✅ **Анонимное прохождение** - участники могут проходить викторины без регистрации
- ✅ Ввод имени участника для статистики (запоминается для повторных прохождений)
- ✅ Шаринг викторин через ссылку или QR-код
- ✅ Подробная статистика и аналитика с таблицей результатов участников
- ✅ Авторизация через Google + Email (опционально)
- ✅ Адаптивный дизайн Material UI
- ✅ Быстрый и легкий (Vite + React)

## Инструкция по установке

### 1. Клонирование и установка зависимостей

```bash
git clone <your-repo-url>
cd quiz-app
npm install
```

### 2. Настройка Supabase

1. Создайте новый проект на [supabase.com](https://supabase.com)
2. Перейдите в Project Settings > API и скопируйте:
   - URL проекта
   - anon/public ключ
3. Перейдите в SQL Editor в панели Supabase
4. Скопируйте и вставьте содержимое файла `supabase-schema.sql` из этого проекта
5. Выполните SQL для создания всех необходимых таблиц и политик

### 3. Настройка переменных окружения

Создайте файл `.env` в корневой директории:

```env
VITE_SUPABASE_URL=ваш_url_проекта_supabase
VITE_SUPABASE_ANON_KEY=ваш_anon_ключ_supabase
```

### 4. Включение аутентификации

В панели Supabase:

1. Перейдите в Authentication > Providers
2. Включите провайдер **Email** (включен по умолчанию)
3. Включите провайдер **Google** (опционально):
   - Создайте OAuth учетные данные в Google Cloud Console
   - Добавьте Google Client ID и Secret
   - Установите redirect URL: `https://ваш-project-ref.supabase.co/auth/v1/callback`

### 5. Запуск сервера разработки

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`

### 6. Деплой на Vercel

1. Загрузите код на GitHub
2. Подключите репозиторий к [Vercel](https://vercel.com)
3. Добавьте переменные окружения в панели Vercel:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
4. Задеплойте!

## Использование

### Создание викторины

1. Войдите или создайте аккаунт
2. Нажмите "Создать викторину" в панели управления
3. Добавьте название викторины, описание и настройте видимость
4. Добавьте вопросы с вариантами ответов
5. Отметьте правильный ответ для каждого вопроса
6. Сохраните и поделитесь!

### Прохождение викторины

1. Просмотрите публичные викторины на главной странице
2. Или используйте общую ссылку/QR-код
3. Отвечайте на вопросы по одному
4. Получайте мгновенную обратную связь по каждому ответу
5. Увидите свой итоговый результат

### Просмотр статистики

1. Перейдите в панель управления
2. Нажмите на иконку статистики у любой викторины
3. Просмотрите:
   - Всего завершенных сессий
   - Средний балл
   - Эффективность по вопросам
   - Графики успешности
   - **Таблицу результатов участников с именами**

### Анонимное прохождение викторин

Участники могут проходить публичные викторины **без регистрации**:

1. Откройте ссылку на викторину или отсканируйте QR-код
2. Введите ваше имя (минимум 2 символа)
3. Пройдите викторину и получите результат
4. Ваше имя и результат появятся в статистике

**Особенности:**
- Имя запоминается в браузере для повторных прохождений
- Авторизованные пользователи автоматически пропускают ввод имени
- Создатель викторины видит всех участников и их результаты
- Полная приватность: достаточно только имени, без email и пароля

## Схема базы данных

Приложение использует следующие таблицы:

- **quizzes**: Метаданные викторины (название, описание, видимость)
- **questions**: Вопросы викторины
- **answers**: Варианты ответов для каждого вопроса
- **quiz_sessions**: Отслеживание попыток прохождения
  - `participant_name` - имя анонимного участника (NULL для авторизованных)
- **session_answers**: Хранение ответов пользователей

См. `supabase-schema.sql` для полной схемы с RLS политиками.

### Миграции

Если вы обновляете существующую базу данных, выполните SQL из папки `migrations/`:
- `add_participant_name.sql` - добавляет поддержку анонимных участников
